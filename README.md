# League of Legends: Rift Tactics

리그 오브 레전드 팬메이드 턴제 전투 프로토타입입니다.

이 프로젝트는 이제 브라우저 UI 중심 MVP보다, 실제로 움직이고 클릭해서 플레이하는 `Pygame` 네이티브 전투 클라이언트를 기본 방향으로 사용합니다.

## 현재 상태

- 기본 플레이 빌드:
  - `native_game`: 아레나형 3 대 3 전투 프로토타입
  - `native_tactics`: 그리드 이동형 전술 실험 빌드
- 현재 흐름:
  - `native_game`: 챔피언 선택 -> 3 대 3 턴제 전투
  - `native_tactics`: 챔피언 선택 -> 시작 배치 -> 3전 원정 전술 전투 -> 보상 -> 전투 요약 -> 경로+런 노드 선택
- 블루 팀은 플레이어 조작, 레드 팀은 AI 조작
- 목표: "게임처럼 보이고 움직이는 전투"를 먼저 완성한 뒤 시스템 확장

## 현재 구현 내용

- 전투 시작 전 챔피언 선택 화면
- 플레이어 9인 풀에서 3인 출전 조합 선택
- 적 9인 풀에서 3인 조합 미리보기 및 재추첨
- 실제 게임 창에서 전장 렌더링
- 챔피언별 실루엣 렌더링과 한국어 폰트 번들
- 실제 챔피언 아이콘을 전장 유닛과 사이드 패널에 적용
- 챔피언 배치, 대기 모션, 시전 모션, 피격 반응
- 투사체, 링 이펙트, 화면 흔들림, 떠오르는 피해 텍스트
- 버튼, 시전, 피격, 보호막, 승패에 대한 기본 효과음과 앰비언트 오디오
- 챔피언 선택과 시전에 반응하는 챔피언별 테마 효과음
- `native_tactics` 챔피언별 전술 특수기 고정과 고유 패시브 2차 차별화
- `native_tactics` 결전 보스 1차와 엘리트 특성 규칙 추가
- `native_tactics` 런 전용 `휴식 거점`, `변수 균열`, `정예 수배` 노드 분기 추가
- `native_tactics` 결전 보스 2차와 보스 전용 목표/각성 연출 추가
- 헤드리스 스크린샷 캡처로 비주얼 결과 확인 가능
- 스킬 버튼 클릭 및 적 유닛 직접 타겟팅
- 속도 기반 턴 순서
- 쿨다운, 보호막, 기절, 간단한 적 AI

## 실행 방법

### 개발 환경에서 바로 실행

처음 한 번만:

```bash
npm run native:setup
```

게임 실행:

```bash
npm run native:play
```

전술 맵 버전 실행:

```bash
npm run tactics:play
```

직접 Python으로 실행:

```bash
.venv/bin/python -m native_game
```

### 리눅스 실행 파일 만들기

로컬 Linux/WSL용 실행 파일 빌드:

```bash
npm run native:package
```

빌드가 끝나면 실행 파일은 `release/rift-tactics`에 생성됩니다.

### 윈도우 `.exe` 받기

- 이 저장소의 GitHub Actions `Build Windows EXE` 워크플로가 `rift-tactics-win.exe`와 `rift-tactics-win.zip` 아티팩트를 생성합니다.
- GitHub 저장소의 `Actions` 탭에서 최신 `Build Windows EXE` 성공 실행을 연 뒤 `Artifacts`에서 받으면 됩니다.
- 저장소에는 `.exe`를 커밋하지 않고, 액션 아티팩트로만 배포합니다.
- 로컬 Windows 환경에서는 `scripts/build-windows.ps1`로 같은 빌드를 재현할 수 있습니다.

### 현재 준비된 배포 형태

- Linux/WSL 실행 파일: `release/rift-tactics`
- Windows 실행 파일: GitHub Actions 아티팩트 `rift-tactics-win.exe`
- WSL/Linux 환경에서는 저장소에 포함된 PulseAudio 런타임 번들로 소리가 나도록 구성했습니다.

## 조작

- 시작 화면에서 블루 팀 챔피언 카드를 눌러 선택 또는 해제
- 우측 `적 재추첨`으로 상대 조합 변경
- `배치 시작`으로 선택한 조합의 시작 위치 조정 단계 진입
- 배치 화면에서 왼쪽 챔피언 카드 선택 후 파란 시작 칸 클릭으로 위치 변경
- 배치 화면에서 `전투 시작`으로 선택한 조합으로 전투 진입
- `native_tactics`는 `정찰전 -> 엘리트전 -> 결전` 3단계 원정 구조
- 각 승리 뒤 보상 하나를 고르고, 전투 요약과 경로 선택을 거쳐 다음 전투 배치 화면으로 이동
- 경로 선택 화면에서는 각 카드가 `경로 + 노드 + 전투 이벤트` 조합으로 제시됨
- 보상 화면에서 다음 적 조합을 미리 확인 가능
- `native_tactics` 전투 화면에서 현재 유닛의 패시브와 적의 이번 행동 + 다음 적 경고 + 적 연속 턴 요약을 함께 확인 가능
- `native_tactics` 챔피언마다 전술용 `특수기`가 고정되어 궁극기/제압기 성격이 일관되게 유지됨
- `native_tactics` 전장에 스테이지별 `수풀`, `룬 지대`, `화염 지대`가 배치됨
- `native_tactics` 2전부터 엘리트 적이 등장하고, 적 의도 패널에 예상 피해, 광역 타겟 수, 위협 칸, 목표 압박이 함께 표시됨
- `native_tactics` 3전은 보스 결전으로 바뀌며, 보스는 체력 절반 이하에서 `결전 각성`을 한 번 발동함
- 엘리트 적은 `철벽`, `맹추격`, `비전 폭주` 중 하나의 특성을 들고 등장함
- 적 의도 패널은 이제 `위험 낮음/보통/높음/치명`, `연쇄 집중`, `처치각`까지 함께 보여 줌
- 전장에는 적 연속 턴의 집중 포커스 대상이 붉은 테두리로 직접 강조됨
- 승리 후 `전투 요약 -> 3안 경로 선택 -> 다음 전투 배치` 흐름으로 이어짐
- 경로 선택은 다음 전투 한정 리스크/보상 보정을 제공하고, AI도 지형/위협뿐 아니라 목표 타일 경쟁까지 고려해 움직임
- 선택한 경로는 전투 중 `맵 목표`를 추가하고, 목표 달성 시 런 강화 1개를 자동으로 지급
- 점령 목표 타일은 전장에 직접 표시되고, 현재 진행도는 전투 HUD에 함께 노출됨
- 각 경로 카드에는 `전투 이벤트`가 같이 붙고, 선택한 이벤트는 다음 전투 한정 추가 효과를 부여함
- 런 노드는 `휴식 거점(안정 회복)`, `변수 균열(이벤트 증폭)`, `정예 수배(추가 정예 + 추가 강화)`로 나뉨
- `변수 균열` 노드를 고르면 경로 이벤트 보너스와 실패 페널티가 함께 커짐
- `휴식 거점` 노드를 고르면 예약된 다음 전투 페널티를 정리하고 회복 보정을 얻음
- `정예 수배` 노드를 고르면 다음 전투의 정예 수가 늘고, 승리 시 런 강화 1개를 추가로 획득함
- 맵 목표를 실패한 채 승리하면 `다음 전투 페널티`가 예약되어 다음 경로/배치 단계까지 이어짐
- 배치 화면과 전투 화면에도 현재 선택한 런 노드 효과가 같이 노출됨
- 결전은 일반 경로 목표 대신 `보스 전용 목표`로 바뀌며, 성공 시 보스 각성이 약화되고 실패 시 더 강해짐
- 보스가 각성하면 주변 칸에 `결전 파동`이 퍼지고, 화면 상단에 결전 배너와 우측 패널 `결전 상태`가 표시됨
- 최종 승리/패배 오버레이에도 결전 목표 결과가 함께 표시됨
- 마우스로 하단 스킬 버튼 클릭
- 단일 대상 스킬은 전장 위 적 챔피언을 클릭해서 사용
- `1`, `2`, `3`: 스킬 선택
- `R`: 전투 재시작
- `ESC`: 전투 중에는 선택 화면으로 복귀, 선택 화면에서는 종료
- 전투 종료 후 오버레이의 `캐릭터 다시 선택`으로 조합 화면 복귀
- 전투 종료 후 `같은 조합 재대결` 또는 `R`로 즉시 재도전
- 최종 승리 후 `같은 조합 새 런`으로 1전부터 다시 시작 가능

## 테스트

전투 로직 테스트:

```bash
npm run native:test
```

전술 엔진 테스트:

```bash
npm run tactics:test
```

헤드리스 스모크 테스트:

```bash
npm run native:smoke
```

전술 맵 스모크 테스트:

```bash
npm run tactics:smoke
```

스크린샷 캡처:

```bash
npm run native:capture
```

전술 맵 스크린샷 캡처:

```bash
npm run tactics:capture
```

패키징된 실행 파일 검증:

```bash
SDL_VIDEODRIVER=dummy SDL_AUDIODRIVER=dummy ./release/rift-tactics --headless --frames 2
```

## 참고

- 기존 `Vite + React + Phaser` 웹 프로토타입도 저장소에 남아 있습니다.
- 하지만 현재 우선순위는 네이티브 `Pygame` 클라이언트를 더 게임답게 만드는 것입니다.
- 챔피언 테마 효과음은 공식 Riot 음원을 가져오지 않고, 각 챔피언 성격을 반영한 오리지널 신스 모티프로 구성했습니다.
- 작업 추적은 `tasks/todo.md`, 교정 기록은 `tasks/lessons.md`, 개발 기록은 `docs/DEVLOG.md`를 사용합니다.
- 전술 UX 설계 근거는 `docs/TACTICS_UX_RESEARCH.md`에 정리했습니다.
- 출시 목표와 완성 기준은 `docs/TACTICS_RELEASE_TARGET.md`를 기준으로 봅니다.
- 단계별 실행 계획은 `docs/TACTICS_MILESTONES.md`에 정리했습니다.
