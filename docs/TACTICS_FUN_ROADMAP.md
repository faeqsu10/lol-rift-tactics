# Tactics Fun Roadmap

`native_tactics`를 "움직이는 게 재밌는 전술 게임"으로 끌어올리기 위한 우선순위 문서.

## 핵심 판단

- 지금 재미의 씨앗은 이미 있다:
  - 이동 + 행동 턴 구조
  - 챔피언별 이름과 역할
  - 맵 위 포지셔닝
- 하지만 오래 붙잡는 요소는 아직 부족하다:
  - 챔피언 고유성
  - 판 전 선택과 배치
  - 지형과 목표
  - 한 판 뒤의 보상

## 전문가 합의

### 전투 설계

- 재미는 "누구를 데려갔는가"보다 "어디에 세웠는가"와 "언제 진입했는가"에서 나와야 함
- 챔피언마다 맵 위 역할이 확실히 달라야 함
- 추천:
  - 고유 패시브
  - 스킬 범위 차별화
  - 위치 기반 콤보

### 시스템 설계

- 전술 게임은 전투 시작 전 선택 단계가 재미를 키움
- 추천:
  - 챔피언 선택
  - 시작 배치
  - 적 조합 미리보기
  - 맵별 추천 조합

### UX 설계

- 플레이어는 난도가 아니라 "이해 불가능함"에서 이탈함
- 추천:
  - 위험 칸 표시
  - 사거리 미리보기
  - 적 의도 표시
  - 예상 피해/기절 미리보기

### 콘텐츠 설계

- 맵이 바닥만 있으면 금방 질림
- 추천:
  - 수풀
  - 벽
  - 버프 타일
  - 전멸 외 승리 조건

### 리텐션 설계

- 한 판 뒤에 작은 성장 보상이 있어야 반복 플레이 이유가 생김
- 추천:
  - 전투 후 1개 보상 선택
  - 런 전용 강화
  - 맵 분기

## 우선순위

### 1단계: 즉시 재미 상승

- `native_tactics` 챔피언 선택 화면
- `native_tactics` 시작 배치 단계
- 챔피언별 고유 패시브 1개씩
- 챔피언별 특수기 범위/효과 차별화

성공 기준:
- 같은 3명 조합이라도 배치와 첫 턴 선택에 따라 결과가 달라짐
- 유저가 "이 챔프는 이런 식으로 써야 한다"를 빠르게 느낄 수 있음

### 2단계: 전술 깊이 확보

- 수풀, 벽, 룬 타일 추가
- 적 의도 표시
- 사거리/범위/예상 피해 하이라이트
- AI가 엄폐, 진입, 마무리를 구분하도록 개선

성공 기준:
- 이동 결정이 단순 접근이 아니라 지형 활용으로 이어짐
- 적 턴이 억울하지 않고 읽을 수 있게 보임

### 3단계: 판 구조 강화

- 거점 점령
- 생존전
- 보스전
- NPC 보호

성공 기준:
- 전멸만이 아니라 맵 목표 때문에 조합 가치가 바뀜
- 맵이 달라지면 정답 플레이도 달라짐

### 4단계: 반복 플레이 루프

- 전투 후 보상 선택 3안 중 1
- 챔피언 강화 노드
- 맵 분기
- 짧은 런 구조

성공 기준:
- 한 판 끝난 뒤 바로 다음 판을 시작할 이유가 생김
- 실패해도 "다음 조합을 다시 해보고 싶다"는 감각이 생김

## 지금 당장 하지 않을 것

- 멀티플레이
- 긴 스토리 모드
- 복잡한 아이템 대량 추가
- 챔피언 수만 무작정 늘리기

## 다음 구현 권장 순서

1. 고유 패시브와 스킬 차별화
2. 지형 3종과 적 의도 표시
3. 맵 분기와 엘리트 전투
4. 전투 후 요약과 장기 보상 구조

## 진행 상태 메모

- 완료:
  - 챔피언 선택
  - 시작 배치
  - 3전 원정 구조
  - 전투 후 보상 선택
  - 전투 후 요약과 경로 선택 1차
  - 경로 선택과 적 의도 다단계 예고 2차
  - 경로별 전용 목표 1차
  - 패시브 1차
  - 적 의도 표시 1차
  - 지형 1차
  - 엘리트전 1차
- 참고:
  - UX 근거와 반영 내용은 `docs/TACTICS_UX_RESEARCH.md`
