# Tactics Release Target

## 목표

`native_tactics`를 단순 프로토타입이 아니라, 반복 플레이가 가능한 싱글플레이 전술 로그라이크 게임으로 완성한다.

## 최종 결과물 정의

- 플랫폼:
  - 1차 목표는 Windows 실행 파일
  - Linux/WSL은 개발과 내부 검증 경로로 유지
- 장르:
  - 짧은 런 구조의 싱글플레이 턴제 전술 게임
- 플레이 길이:
  - 한 런 기준 20분에서 35분
- 핵심 루프:
  - 챔피언 선택 -> 시작 배치 -> 전술 전투 -> 보상 -> 경로 선택 -> 이벤트/엘리트/보스 -> 런 종료

## 게임 기둥

### 1. 읽히는 전술

- 적 의도, 예상 피해, 위험 칸, 목표 진행도가 전부 화면에서 읽혀야 한다.
- 플레이어는 "몰라서 맞는" 느낌보다 "읽고도 감수하는" 느낌을 받아야 한다.

### 2. 챔피언 개성

- 챔피언은 이름만 다른 유닛이 아니라 전술 역할이 완전히 달라야 한다.
- 데려가는 조합에 따라 첫 턴, 진입 타이밍, 지형 활용 방식이 달라져야 한다.

### 3. 런 빌드 다양성

- 전투 후 보상, 경로 선택, 맵 목표 보상 때문에 같은 챔피언 조합도 매번 다른 성장 경로를 타야 한다.
- 실패해도 "이번엔 다른 빌드로 다시 해볼까"가 나와야 한다.

### 4. 짧지만 진한 완주감

- 한 판이 질질 늘어지지 않아야 한다.
- 초반 정찰전, 중반 엘리트전, 후반 결전이 확실히 다른 긴장감을 줘야 한다.

## 1차 완성 기준

아래를 만족하면 `1차 완성`으로 본다.

- 플레이어가 처음부터 끝까지 한 런을 문제 없이 완주 가능
- 챔피언 선택, 배치, 전투, 보상, 경로, 이벤트, 보스까지 한 사이클이 연결됨
- 최소 8명 이상의 플레이어 챔피언과 8명 이상의 적 챔피언이 의미 있게 차별화됨
- 맵 목표와 경로 이벤트가 최소 6종 이상 존재
- 보스전이 최소 2종 존재
- 적 의도 UI가 다중 타겟 또는 연쇄 위협까지 읽히게 제공됨
- 기본 사운드와 전투 연출이 "프로토타입"이 아니라 "출시 후보" 수준으로 정리됨
- 윈도우 실행 파일을 외부 플레이테스터에게 배포 가능한 상태

## 출시판에서 꼭 들어가야 하는 것

- 짧은 튜토리얼 또는 온보딩 문구
- 난이도 2단계 이상
- 설정:
  - 볼륨
  - 해상도 또는 화면 크기
  - 키 안내
- 런 종료 요약
- 패배 뒤 즉시 재도전 동선
- 최소한의 저장:
  - 설정 저장
  - 최고 기록 또는 최근 런 기록

## 출시판에서 빼는 것

- 멀티플레이
- 긴 스토리 캠페인
- 과도한 챔피언 수집형 구조
- 챕터 수가 많은 대형 메타 진행
- 웹 빌드 재우선화

## 성공 기준

- 외부 플레이어가 설명 없이도 1런을 진행할 수 있다.
- 플레이어가 3런 이상 반복 플레이할 이유를 느낀다.
- 패배 원인을 UI만 보고 어느 정도 설명할 수 있다.
- 조합/경로/보상 선택 때문에 "이번 판이 지난 판과 달랐다"는 감각이 있다.

## 현재 판단

- `native_game`은 보조 프로토타입으로 유지
- 실질적 출시 후보는 `native_tactics`
- 따라서 앞으로의 큰 설계 결정은 `native_tactics` 기준으로만 내린다.
