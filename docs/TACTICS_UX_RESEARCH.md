# Tactics UX Research

## 목적

`native_tactics`가 "한 판 하고 끝나는 실험 화면"이 아니라, 사용자가 다음 행동을 자연스럽게 고르고 반복 플레이하게 만드는 전술 게임 루프를 정의한다.

## 참고한 1차 출처

1. Riot Games, `Clarity in League`
   - https://www.riot.com/en/news/clarity-in-league
   - 핵심: 게임 상태와 위험 요소는 플레이어가 빠르게 읽을 수 있어야 하며, 시각적 소음보다 중요한 정보 전달이 우선이다.

2. Subset Games, `Into the Breach`
   - https://subsetgames.com/itb.html
   - 핵심: 적 행동이 미리 보이고, 플레이어는 완전 정보에 가까운 상황에서 전술 결정을 내릴 때 재미를 느낀다. 공식 소개도 "every enemy attack is telegraphed"와 파일럿/무기 확장을 핵심 가치로 둔다.

3. Mega Crit, `Slay the Spire` 공식 소개
   - https://store.steampowered.com/app/646570/Slay_the_Spire/
   - 핵심: 매 전투 뒤 달라지는 강화 선택과 다음 경로를 고르는 구조가 반복 플레이 동기를 만든다.

4. Ryan, Rigby, Przybylski, `The Motivational Pull of Video Games`
   - https://pmc.ncbi.nlm.nih.gov/articles/PMC8829687/
   - 핵심: 자율성, 유능감, 관계성 같은 심리 욕구 충족이 게임 몰입과 선호에 강하게 연결된다.

## 우리 게임에 대한 해석

- 추론: 지금 `native_tactics`의 가장 큰 문제는 전투 자체보다 "전투 뒤 다음 목표가 비어 있는 것"이었다.
- 추론: 전술 게임에서 반복 플레이를 만들려면 아래 네 가지가 동시에 필요하다.
  - 명확한 다음 목표
  - 눈에 보이는 성장
  - 다음 판에서 달라지는 선택
  - 패배/성공 뒤 분명한 재진입 버튼

## 반영한 결정

### 1. 3전 원정 구조

- 선택 화면에서 바로 단발성 1회 전투가 아니라 `정찰전 -> 교전 -> 결전` 3단계 원정으로 본다.
- 이유:
  - 전투마다 맥락이 생긴다.
  - "이번 판만 이기면 끝"이 아니라 "이번 승리가 다음 전투를 바꾼다"는 기대가 생긴다.

### 2. 전투 후 보상 선택

- 중간 전투에서 승리하면 바로 다음 전투로 넘기지 않고 보상 하나를 고른다.
- 현재 보상 예시:
  - 체력 증가
  - 피해 증가
  - 속도 증가
  - 이동력 증가
  - 시작 보호막
- 이유:
  - 자율성: 플레이어가 직접 다음 전투의 성격을 고른다.
  - 유능감: 같은 조합도 점점 강해지는 체감을 만든다.

### 3. 다음 적 조합 미리보기

- 보상 화면에서 다음 적 조합을 먼저 보여 준다.
- 이유:
  - 사용자가 "왜 이 보상을 골라야 하는지" 판단할 근거를 제공한다.
  - 단순 랜덤 강화보다 전략 선택으로 느껴진다.

### 4. 전투 종료 버튼 명시

- 전투 종료 시 `캐릭터 다시 선택`, `같은 전투 재도전`, `같은 조합 새 런` 같은 다음 행동을 버튼으로 직접 제공한다.
- 이유:
  - 숨은 단축키만 남기면 흐름이 끊긴다.
  - 실패 이후 재진입 비용을 낮춰야 반복 플레이가 생긴다.

## 다음 우선순위

1. 적 의도 표시
   - `Into the Breach`처럼 "이번 턴 적이 누구를 노리는지"를 UI에 직접 보여주기

2. 엘리트/이벤트 분기
   - `Slay the Spire`처럼 모든 전투가 같은 가치가 되지 않도록 분기 추가

3. 챔피언별 패시브
   - 지금 보상은 전역 강화에 가깝다.
   - 다음 단계는 챔피언 고유 전술성을 강화해야 한다.

4. 패배 요약
   - 어떤 선택이 통했고 어떤 순간에 무너졌는지 짧게 요약해 주기
