## 현재 상태 진단

### 핵심 문제

1. **해상도/스케일링**: 1600x960 고정 (16:10), fullscreen/resize 미지원
2. **절대 좌표 의존**: 패널/버튼/텍스트 위치가 거의 전부 픽셀 하드코딩
3. **색상 비일관**: 팔레트 상수 120회 vs 하드코딩 RGB ~444개 (78.7%가 하드코딩)
4. **폰트 계층 미약**: font_micro(12px)와 font_tiny(13px) 사이 1px 차이, 핵심 전투 정보가 13px로 표시
5. **컴포넌트 재사용 부족**: 공용 함수 4개 외에 각 화면에서 인라인 패널/버튼 렌더링 반복
6. **호버 효과 완전 부재**: MOUSEMOTION 처리 없음, 클릭 가능 여부 시각 구분 불가
7. **ESC 키 무확인 동작**: 런 진행 중 ESC로 즉시 리셋, select에서 ESC로 게임 즉시 종료

### 정밀 수치 현황 (전문가 분석 기반)

| 항목 | 현재 값 | 목표 |
|---|---|---|

### 타이포그래피

**현재 → 제안 (7단계 → 6단계)**

| 현재 | 크기 | 문제 | 제안 | 크기 | 용도 |
|---|---|---|---|---|---|
| font_title | 42px bold | OK | font_display | 44px bold | 오버레이 타이틀, 승리/패배 |
| font_large | 34px bold | 3회만 사용 | *(삭제)* | - | font_heading으로 통합 |
| font_heading | 26px bold | OK | font_heading | 28px bold | 패널 제목, 서브 헤딩 |

## 개편 단계

### Phase 0: 긴급 UX 수정 (선행 작업)

**목표:** 플레이어 경험을 파괴하는 치명적 문제 즉시 해결

| # | 항목 | 상세 | 규모 |
|---|---|---|---|
| 0-1 | **ESC 확인 다이얼로그** | select: "게임을 종료하시겠습니까?", 런 중: "원정을 포기하시겠습니까?" | 소 |
| 0-2 | **전투 승리 후 Enter 키 수정** | 중간 스테이지 승리시 Enter → reward(다음 단계)로. 현재는 select로 가서 런이 끊김 | 소 |
| 0-3 | **호버 효과 기초** | 모든 버튼/카드에 MOUSEMOTION 기반 hover 피드백 (테두리 밝기 +20%, 배경 밝기 +8%) | 중 |
| 0-4 | **지형 범례 상시 표시** | 배치/전투 그리드 옆에 지형 종류별 색상+이름+효과 범례 | 소 |

### Phase 1: 해상도 + 레이아웃 엔진

**목표:** 이후 모든 화면 작업의 토대

1. **해상도 시스템 도입**
   - 기본 해상도를 1920x1080 (16:9)으로 변경
   - 내부 렌더링은 기본 해상도 기준 → `pygame.transform.smoothscale`로 실제 윈도우 크기에 맞게 스케일링
   - F11 풀스크린 토글 추가
   - 설정에 해상도 선택지 추가 (1280x720, 1600x900, 1920x1080)

2. **레이아웃 상수 리팩터링**
   - 전투/배치 모드 패널 조정:
     - LEFT_PANEL: 284px → **320px** (전투 정보 가독성 향상)
     - GRID_CELL: 100px → **92px** (그리드 736px)
     - RIGHT_PANEL: 384px → **412px** (결전 정보 공간 확보)
     - BOTTOM_PANEL: 92px → **120px** (목표/상태 2줄 표시 가능)
   - 비율 기반 계산 함수로 전환
   - 여백/패딩 상수 표준화

3. **폰트 시스템 정비**
   - 6단계 체계 적용 (위 타이포그래피 명세 참조)
   - 폰트 크기를 해상도 비율에 맞게 동적 계산
   - `_draw_text_fit` fallback 체인 단순화 (2단계 → 1단계)

### Phase 2: 색상 + 컴포넌트 통일

### Phase 3: 화면별 레이아웃 재설계

#### 3-1. 챔피언 선택 화면

**현재 문제** (전문가 분석):
- 후보 로스터 11명이 4열 3행에서 카드 ~210x100px, 초상화 60px로 존재감 부족
- 교리 패널이 첫 런에서도 공간 차지 (잠김 상태에서 숨기기 필요)
- 적 프리뷰 영역이 우측 하단에 밀려 좁음
- 스탯 비교 도구 부재

**개선안** (LoL 챔피언 선택 + TFT 상점 참조):
- 후보 로스터: 4열 → **3열** (카드 폭 ~280px, 초상화 72px으로 확대)
- 포커스 패널: 스탠디(172x208px) 유지, 스탯 비교 기능 추가
- 첫 런에서 교리 패널 축소/숨기기
- 출전 슬롯과 후보 카드의 시각적 계층 차이 강화

#### 3-2. 배치 화면

**현재 문제**:
- LEFT_PANEL 284px에 챔피언 3장 + 배치 규칙 + 노드 정보 → 공간 압박
- BOTTOM_PANEL 92px에 버튼 + 상태 + 노드 효과 → 텍스트 겹침 위험
- 배치 시작 칸 색상 하드코딩

**개선안** (Into the Breach 유닛 배치 참조):
- 좌측 패널 320px로 확대 (Phase 1과 연동)
- 하단 패널 120px로 확대
- 지형 범례를 그리드 옆에 상시 표시
- 배치 가능 영역에 펄스 하이라이트 추가

#### 3-3. 전투 화면

**현재 문제** (전문가 분석):
- 좌측 284px에 활성 유닛 + 패시브 + 적 의도 + 조작 가이드 → 빽빽함
- 적 의도가 텍스트 기반 (Into the Breach는 그리드 오버레이)
- 목표 진행도가 하단 한 줄 → 중요도 대비 가시성 매우 낮음
- 플로팅 텍스트와 유닛 스탠디 겹침
- 하단 4버튼에 호버 효과 없음

**개선안** (Into the Breach 적 의도 + Slay the Spire 레이아웃):
- 적 의도를 **그리드 위 시각 오버레이**로 전환 (빨간 타일 + 화살표 + 피해 수치)
  - Into the Breach 원칙: "텍스트 설명보다 시각 오버레이 우선"
- 좌측 패널에서 적 의도 텍스트 제거 → 활성 유닛 정보 + 패시브에 집중
- 목표 진행도를 그리드 상단 또는 헤더 영역으로 이동
- 턴 순서를 우측 상단에 초상화 바 형태로 표시

#### 3-4. 보상 화면

**현재 문제**:
- 적절한 카드 크기(538x160px)이나 하단에 ~80px 빈 공간
- 다음 적 조합 영역에 하단 ~66px 빈 공간

**개선안** (Slay the Spire 카드 보상):
- 보상 카드에 호버 시 확대 효과
- 건너뛰기(Skip) 옵션 명시
- 보상 효과 미리보기 강화

#### 3-5. 경로 선택 화면

**현재 문제** (전문가 분석):
- 경로 카드 3장에 각 6~7줄 텍스트 → 밀도 높음
- 하단 버튼 3개가 538px에서 촉박
- 좌측 프리뷰에 6종 정보 동시 노출 → 비교 부담

**개선안** (Slay the Spire 맵 + Into the Breach 임무 선택):
- 경로 카드 정보를 2단계로 분리: 요약(아이콘+한줄) / 상세(좌측 프리뷰)
- 카드에 노드 구성 아이콘 미리보기
- 보상/위험을 색상+아이콘으로 시각 구분 강화

#### 3-6. 결산 화면

**현재 문제**:
- 좌측 서브패널이 균등하게 작아 시각적 앵커 부재
- 1~2전 패배 시 우측 타임라인 카드가 1~2개뿐 → 하단 빈 공간
- 패널 테두리가 동일 골드색 → 단조로움

**개선안** (Darkest Dungeon 원정 결과 + TFT 라운드 결과):
- 성공/실패에 따라 accent 색상을 패널 자체에도 반영
- 런 타임라인을 수평 배치로 전환 (상단 영역)
- MVP 챔피언 하이라이트 연출

### Phase 4: 폴리시 + 전환 효과

1. **화면 전환 애니메이션** — 페이드(0.3초) + 슬라이드
2. **호버 효과 강화** — 카드 호버 시 미세 확대(1.02x), 버튼 호버 시 배경 밝기 변화
3. **선택 피드백** — 선택 시 0.15초 스케일 펄스 (1.0 → 1.05 → 1.0)
4. **전투 인트로 개선** — 결전 카드 타이머 1.65초 → 2.0초, 모티프 장식 유지
5. **승리/패배 오버레이 개선** — 라운드 수, 핵심 스탯 포함

## 전문가 분석 결과 요약

### Visual Design Audit (비주얼 디자인 감사)

**수직 예산 분석** (960px):
- 상단 여백 28px + 헤더 72px + 메인 694~784px + 하단 92px + 하단 여백 36px
- BOTTOM_PANEL 92px가 전투 버튼(46px) + 여백으로 꽉 참 → 120px 확장 필요

**수평 예산 분석** (1600px):
- 전투: LEFT 284px + 갭 40px + GRID 800px + 갭 20px + RIGHT 384px + 여백 72px
- LEFT_PANEL 284px는 전투 정보 담기에 부족 → 320px 확장 권고

**컴포넌트 품질**:
- 공유 함수 4개(`_draw_panel`, `_draw_battle_card`, `_draw_message_strip`, `_draw_info_chip`) 외 인라인 패널 렌더링 다수
- 같은 골드 활성/비활성 버튼 패턴이 ~30곳에서 인라인 반복
- border_radius 253회 사용 중 9종 변형 → 4종으로 표준화

**화면별 핵심 발견**:
- 선택: 후보 카드 60px 초상화 → 존재감 부족, 4열 → 3열 권고
- 배치: 지형 범례가 하단 한 줄뿐, 배치 가능 영역 시각 안내 부족
- 전투: 좌측 패널에 유닛+패시브+적 의도+조작가이드 과밀, 호버 0개
- 보상: 적절한 밀도이나 하단 빈 공간 존재
- 경로: 카드 내부 텍스트 6~7줄로 밀도 높음, 버튼 3개 간격 촉박
- 결산: 서브패널 균등 크기로 시각 앵커 부재

### UX Flow Analysis (UX 흐름 분석)

**치명적 UX 버그**:
1. **ESC 무확인 런 포기** — 전투/보상/경로 중 ESC → 확인 없이 즉시 select 복귀, 모든 런 진행 초기화
2. **ESC 무확인 게임 종료** — select 화면에서 ESC → 확인 없이 `self.running = False`
3. **전투 승리 후 Enter 버그** — 중간 스테이지 승리 오버레이에서 Enter → `_return_to_select()` 호출, 자동 전환(1.2초 후 reward)보다 먼저 누르면 런 끊김

**호버/커서 피드백 완전 부재**:
- MOUSEMOTION 이벤트 미처리
- 마우스 커서 기본 화살표 고정
- 클릭 가능/불가능 구분이 오직 색상/위치에 의존

**온보딩 문제**:
- 지형 효과 수치 미상세 노출 (보호막 +8, 피해 3 등)
- 턴제 전투 규칙(속도 기반 턴 순서, 쿨다운) 미설명
- 챔피언 간 스탯 비교 기능 없음
- 교리 시스템이 첫 런에서 공간만 차지

**정보 중요도 vs 가시성 불일치**:

| 정보 | 중요도 | 가시성 | 문제 |
|---|---|---|---|
| 지형 효과 | 높음 | 매우 낮음 | 하단 한 줄 |
| 적 의도 | 높음 | 중간 | 좌측 패널 하단 13px 텍스트 |
| 턴 순서 | 높음 | 낮음 | 우측 상단 한 줄 |
| 목표 진행 | 높음 | 낮음 | 하단 패널 한 줄 |
| 교리 | 낮음(첫런) | 중간 | 우측 고정 영역 |
