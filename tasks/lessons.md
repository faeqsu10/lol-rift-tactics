# Lessons

## 2026-03-09

- 사용자가 "게임같이"를 요구하면 텍스트 UI나 카드형 웹 화면이 아니라 실제 전장 감각을 우선 기준으로 잡을 것
- Linux 실행 환경에서는 한국어 시스템 폰트가 없을 수 있으니 폰트를 저장소에 번들하는 쪽을 먼저 고려할 것
- 비주얼 피드백 요청에서는 로직 구현보다 스크린샷 검증 경로를 먼저 준비할 것
- 사용자 요청에 따라 작업 단위가 생길 때마다 커밋과 푸시를 바로 수행할 것
- 적 대상 스킬의 유효 타겟 조회는 실제 시전뿐 아니라 미리보기와 클릭 하이라이트에서도 같은 결과를 반환해야 함
- 플레이 가능한 전투를 만들었으면 효과음이 없더라도 "완성"으로 보지 말고 기본 오디오 피드백을 함께 넣을 것
- WSL/Linux 배포에서는 Python 패키지만으로 끝나지 않고 오디오 런타임 라이브러리 누락 여부도 같이 확인할 것
- 사용자가 "게임 같다"는 감각을 원하면 전투 화면뿐 아니라 시작 화면, 선택 흐름, 재진입 흐름까지 같이 설계할 것
- 챔피언 선택 풀은 금방 늘어나므로 카드 레이아웃을 고정 수치로 두지 말고 풀 개수에 따라 열과 높이를 계산하도록 만들 것
- `PyInstaller`로 패키징할 때 패키지 내부 `__main__.py`를 직접 진입점으로 쓰면 상대 import가 깨질 수 있으니, 루트 런처 파일을 따로 두는 편이 안전함
- Riot 공식 게임 음원을 바로 쓰기보다, 테마를 닮은 오리지널 사운드를 합성해서 넣는 쪽이 법적 리스크와 유지보수 면에서 낫다
- Windows `.exe`는 Linux/WSL에서 억지로 교차 빌드하지 말고, Windows 러너나 실제 Windows 환경에서 PyInstaller를 돌리는 경로를 기본값으로 둘 것
- 챔피언을 늘릴 때는 아이콘, 전투 데이터, 선택 화면, 사운드 프로필, 테스트를 같이 묶어 업데이트해야 회귀가 적다
