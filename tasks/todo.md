## 리뷰 메모

- 현재 목표는 "게임처럼 보이는가"를 우선 해결하는 것
- 완성 기준과 우선순위는 `docs/TACTICS_RELEASE_TARGET.md`, `docs/TACTICS_MILESTONES.md`를 기준으로 관리
- 도형 프로토타입 느낌을 줄이기 위해 캐릭터 실루엣과 전장 레이어를 먼저 강화함
- 스크린샷 캡처 커맨드는 `npm run native:capture`
- 전술 실험은 기존 아레나 버전 위에 덮어쓰지 말고 별도 패키지로 병행 유지
- `native_tactics`의 다음 재미 포인트는 챔피언 수 확장보다 선택, 배치, 지형, 의도 표시, 보상 루프 순으로 붙이는 것이 효율적
- `native_tactics`처럼 상태가 여러 단계로 나뉘는 화면은 단순 프레임-0 스모크만으로 끝내지 말고 상태 전환까지 직접 검증할 것
- 전술 게임의 중독성은 전투 자체뿐 아니라 전투 뒤 다음 목표 제시에서 크게 갈리므로, 승리 후 보상과 다음 적 프리뷰를 먼저 붙이는 것이 효과적
- 적 의도 표시는 단순 문구만이 아니라 전장 위 하이라이트까지 같이 보여 줘야 플레이어가 바로 이해한다
- 지형 타일은 설명 문구만 두지 말고 전장 색, 아이콘, 전투 효과가 함께 묶여 있어야 플레이어가 빠르게 학습한다
- 전투 뒤 선택 화면이 늘어날수록 흐름은 `보상 -> 경로 -> 배치`처럼 한 단계씩 명확히 나누는 편이 UI와 테스트 모두 안정적이다
- 3안 선택 화면은 카드 수를 늘리는 순간 버튼과 겹치기 쉬우므로, 기능 추가와 동시에 레이아웃 밀도까지 같이 검증할 것
- 전투 중 항상 보여야 하는 목표 정보는 하단 HUD에 한 줄 요약으로 넣는 편이 박스 하나를 더 늘리는 것보다 안정적이다
- 전술 AI가 목표를 노리는 기능은 앱 상태와 엔진 상태가 따로 놀기 쉬우므로, 목표 타일 동기화를 별도 헬퍼로 고정하는 편이 안전하다
- 전투 후 경로 선택이 실제 전략이 되려면 카드 안에 `이번 전투 이벤트`와 `실패 시 대가`가 함께 보여야 한다
- 챔피언별 스킬 성격이 중요한 전술 모드에서는 `특수기`를 휴리스틱으로 고르지 말고 명시 매핑으로 고정하는 편이 안전하다
- 결전 스테이지는 일반 엘리트 강화만 재활용하지 말고 `보스`와 `엘리트 부관`을 분리해서 표식과 규칙을 따로 주는 편이 체감이 훨씬 좋다
- 적 의도 고도화는 총합 피해 숫자만 늘리지 말고 `집중 포커스 대상`, `치명 위험도`, `연쇄 집중 문구`를 같이 보여 줘야 실제 판단에 도움이 된다
- 선택 화면 프리뷰와 실제 전투 적용 수치가 달라지면 바로 신뢰가 깨지므로, 노드/이벤트처럼 겹치는 보정은 같은 계산 함수를 UI와 로직에서 함께 써야 한다
- 최종전 규칙을 별도로 추가하면 경로 화면과 배치 화면의 프리뷰도 같은 규칙으로 바꿔 줘야 사용자가 마지막 전투에서 속았다고 느끼지 않는다
- 런 종료 UX는 전투 승패 오버레이에 계속 덧대기보다 별도 `summary` 상태 화면으로 분리하는 편이 입력 처리와 회귀 테스트가 훨씬 안정적이다
- 경로 카드에 노드 하위 이벤트까지 붙이면 카드 밀도가 급격히 올라가므로, 캡처로 줄 수를 확인하면서 카드와 상세 패널 사이에 정보를 분산해야 한다
- 배치 화면에서 선택 화면으로 빠지는 입력은 단순 `screen_mode` 변경으로 끝내지 말고, 항상 공통 런 초기화 경로를 타게 해야 상태 누수를 막을 수 있다
- 결전 스테이지를 늘릴 때는 목표 문구만 바꾸지 말고 `지형`, `장애물`, `보스 패턴`, `프리뷰 문구`를 한 묶음으로 바꿔야 실제로 다른 보스전처럼 체감된다
- 결산 화면에 영속 기록을 붙일 때는 테스트와 스모크가 파일 하나를 공유하지 않게 `headless` 기본 저장을 끄고, 필요한 테스트만 임시 경로를 주입하는 편이 안전하다
- 전투 시작 연출은 결전 배너와 한 화면에서 경쟁하므로, `상단 배너`와 `중앙 컷인`처럼 역할을 분리해 겹치지 않게 두는 편이 읽기 쉽다
- 보스 각성에 추가 피해 규칙을 붙일 때는 일반 스킬 타격과 같은 `impacts` 배열을 그대로 재사용하지 말고, 스킬 본타와 각성 부가 피해를 분리해 패시브 트리거 오염을 막아야 한다
- 메타 진행은 결산 화면 안쪽에만 숨기지 말고, 다음 런 시작 전 선택 화면에서 직접 고를 수 있는 형태로 꺼내야 반복 플레이 동기가 실제로 생긴다
- 전투 HUD는 카드 좌표를 고정값으로 멀리 떨어뜨리기보다 `이전 카드 bottom + 여백`으로 계산해야 패널 높이 조정 때 텍스트 겹침이 재발하지 않는다
- 실제 스프라이트가 아직 없어도, 초상화 + 역할 액세서리 + 팀색 배너를 조합한 `스탠디` 렌더만으로 도형 프로토타입 느낌을 크게 줄일 수 있다
- 전투 시작 컷인은 제목만 바꾸지 말고 `모티프 도형 + 배지 + 사운드`를 한 세트로 묶어야 노드/결전 성격이 즉시 인식된다
- 승리 직후 바로 보상/결산 화면으로 넘기면 승리 포즈가 보이지 않으므로, 짧은 전투 종료 홀드 타이머를 두는 편이 체감 품질이 훨씬 좋다
- 결전 변형이 이미 데이터로 나뉘어 있다면, 전장 팔레트와 장애물 미술도 같은 변형 키를 따라가게 해야 보스전 체감이 훨씬 또렷해진다
